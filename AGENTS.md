# Agent Rules

在这个仓库工作时，请遵循 [RULES.md](RULES.md)。

开始任何开发、脚本或交付任务前，也请先阅读 [CODING_PROMPTS.md](CODING_PROMPTS.md)，并把其中的上下文作为项目提示词参考。

## 工作方式

- 先理解目标和范围，再动手修改。
- 保持改动小而清晰，不做无关重构。
- 新增规则、依赖或结构时，说明它解决的实际问题。
- 如果项目只有五天节奏，优先保证主路径闭环和可交付状态。

## 提交前检查

- 检查 `git status`，只提交与任务相关的文件。
- 不提交密钥、个人配置、临时文件或生成垃圾。
- 如果项目后来添加了测试、lint 或构建命令，提交前运行对应检查。
